คู่มือใช้งาน Stock2 (ทดสอบระบบแล้ว)

เอกสารฉบับนี้จัดทำจากการทดสอบระบบจริงที่ http://localhost:888/stock2/ พร้อมภาพหน้าจอ Web GUI จริง

วันที่ทดสอบ: 14 April 2026 ผู้ใช้ทดสอบ: 222 / 222 โฟลเดอร์ภาพ: docs/manual_images

สารบัญ

  1. ขอบเขตและผลการทดสอบ
  2. วิธีเริ่มใช้งานระบบ
  3. คำอธิบายแต่ละหน้าจอพร้อมรูป
  4. ตัวอย่างลำดับการใช้งานจริง
  5. การแก้ปัญหาเบื้องต้น

1) ขอบเขตและผลการทดสอบ

ทดสอบผ่านสคริปต์ระบบ (Smoke Tests) และหน้า Web GUI หลัก

2) วิธีเริ่มใช้งานระบบ

  1. เปิดเว็บ http://localhost:888/stock2/
  2. ล็อกอินด้วย Username และ Password ที่ได้รับสิทธิ์ใช้งาน
  3. หลังล็อกอิน ให้เลือกเมนูจาก Sidebar ด้านซ้าย
  4. สำหรับงาน CRUD ให้เข้า ?page=module&module=... ตามโมดูลที่ต้องการ

บัญชีตัวอย่างจากฐานข้อมูลเดิม: 222 / 222

3) คำอธิบายแต่ละหน้าจอพร้อมรูป

3.1 หน้า Login

ใช้ยืนยันตัวตนก่อนเข้าระบบ ถ้าข้อมูลไม่ถูกต้องจะเข้า Dashboard ไม่ได้

หน้าล็อกอินของระบบ Stock2

3.2 หน้า Dashboard

เป็นหน้าสรุปภาพรวมระบบและจุดเริ่มต้นก่อนเข้าโมดูลเฉพาะงาน

Dashboard หลังล็อกอินสำเร็จ

3.3 หน้า Module (ตัวอย่าง: Sale Cash)

เป็นหน้าจอทำงานหลักสำหรับจัดการข้อมูลแบบ CRUD (Add / Edit / Delete / Search / Export)

  • ปุ่ม Add Main สำหรับเพิ่มข้อมูลหัวเอกสาร
  • ปุ่ม Refresh โหลดข้อมูลใหม่
  • ปุ่มรายงานเพื่อเปิดมุมมองเอกสาร/รายการ
หน้าจอโมดูล sale_cash

3.4 หน้า Report (Sale Cash List)

ใช้ดูรายงานรายการเอกสารในรูปแบบที่พิมพ์หรือส่งออกได้

รายงานแบบรายการของโมดูล sale_cash

3.5 หน้า Department Access

ใช้กำหนด/ตรวจสอบสิทธิ์ตามแผนก ว่าแต่ละแผนกเข้าถึงหน้าจอไหนได้บ้าง

หน้าจัดการสิทธิ์ตามแผนก

3.6 หน้า Admin Access

ใช้บริหารสิทธิ์รายผู้ใช้ เช่น View / Add / Edit / Delete / Report ตามโมดูล

หน้าจัดการสิทธิ์ระดับผู้ดูแล

3.7 หน้า ERP Flow

ใช้ดูและทดสอบ flow ทางธุรกิจแบบ end-to-end ระหว่างโมดูล

หน้าจอ ERP Flow

3.8 หน้า Capture Screen

ใช้อัปโหลดรูปหลักฐานหน้าจอการทำงาน พร้อม metadata (module/stage/department/doc ref)

หน้าจอ Screen Capture Log

3.9 หน้า Business Report (GL Trial Balance)

หน้ารายงานสายบัญชี/การเงิน สำหรับตรวจสอบผลรวมเดบิตเครดิตตามช่วงเวลา

รายงาน GL Trial Balance

4) ตัวอย่างลำดับการใช้งานจริง (แนะนำ)

  1. ล็อกอินเข้าระบบ
  2. ตรวจสิทธิ์จาก Department Access / Admin Access (เฉพาะผู้ดูแล)
  3. เข้าโมดูลงาน เช่น sale_cash เพื่อทำ CRUD เอกสาร
  4. เปิด report.php เพื่อตรวจผลรายงาน
  5. ถ้าต้องเก็บหลักฐานขั้นตอน ให้เปิด capture_screen.php และอัปโหลดภาพพร้อมหมายเหตุ

5) การแก้ปัญหาเบื้องต้น

  • เข้าเว็บไม่ได้: ตรวจ Apache/XAMPP ว่ารันอยู่ และ URL ถูกต้อง
  • ล็อกอินไม่ได้: ตรวจ username/password และสิทธิ์ในฐานข้อมูล
  • หน้าโมดูลว่าง: ตรวจสิทธิ์ผู้ใช้ใน user_access / user_module_access
  • รายงานไม่ขึ้น: ตรวจว่ามีข้อมูลในตารางต้นทาง และช่วงวันที่ถูกต้อง

หมายเหตุ: สิทธิ์แต่ละบัญชีอาจเห็นเมนูไม่เหมือนกัน
