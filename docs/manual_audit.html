Role Objective

Independently verify process compliance and maintain evidence package from operation screens and reports.

Daily Checklist

Start of Day

  1. Review audit plan and critical transactions for the day.
  2. Check rights changes and high-risk module activity.
  3. Prepare evidence naming and reference standards.

Operations

  1. Validate transaction trace from source to report output.
  2. Capture key screens with stage code and document reference.
  3. Cross-check warehouse, production, and accounting consistency.
  4. Log nonconformities with reproducible screenshot evidence.

Close of Day

  1. Compile evidence set by project/run and department.
  2. Issue finding summary with severity and owner.
  3. Verify corrective actions and reopen unresolved critical points.

Audit Evidence To Capture

  1. Permission matrix and affected user scope.
  2. Process timeline screen with linked document references.
  3. Financial and inventory report snapshots for reconciliation.

Use capture_screen.php and tag each screenshot with module_key, process_stage, project_code, run_no, and doc_ref.

Role KPIs

  • Audit coverage of critical steps = 100%.
  • All critical findings have owner and target date.
  • Evidence package completeness pass before sign-off.
